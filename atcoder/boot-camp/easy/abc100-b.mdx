## Explanation

We want the $N$-th smallest positive integer that can be divided by $100$ **exactly** $D$ times.

"Divisible by 100 exactly $D$ times" means:

- the number is divisible by $100^D$
- but **not** divisible by $100^{D+1}$

### Key observation

Any number that is divisible by $100$ exactly $D$ times can be written as:

$$
( x \cdot 100^D )
$$

where $x$ is a positive integer **not divisible by 100**.

So, if we list valid numbers in increasing order, it is the same as listing $x$ in increasing order while skipping multiples of $100$, and then multiplying by $100^D$.

### What happens at $N = 100$?

If we naively take $x = N$, then at $N = 100$ we would use $x = 100$, but $100$ is divisible by $100$.
That would make:

$$
100 \cdot 100^D = 100^{D+1}
$$

which is divisible by $100$ **at least** $D+1$ times (too many).

Therefore, the sequence of allowed $x$ values is:

- $1, 2, 3, \dots, 99, 101, 102, \dots$

So:

- if $N \neq 100$, use $x = N$
- if $N = 100$, use $x = 101$

Then answer is $x \cdot 100^D$.

## Implementation

**Time Complexity:** $O(1)$

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	long long d, n;
	cin >> d >> n;

	long long power = 1;
	for (long long i = 0; i < d; i++) {
		power *= 100;
	}

	long long base = (n == 100 ? 101 : n);
	cout << base * power << '\n';
	return 0;
}
```
